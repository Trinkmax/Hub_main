## Verificación 3: smoke owner end-to-end

Levantá el dev server:
```bash
npm run dev
```

Logueate como **owner del tenant seed** (ver `supabase/seed.sql` para credenciales) y andá a `http://localhost:3000`.

### 3.1 Crear, editar, regenerar, eliminar mesas

1. Navegá a `/<slug>/configuracion/mesas`. Verificá que la página carga con título "Mesas" y botón "Nueva mesa".
2. Tocá "Nueva mesa". Form con campos `Nombre` y `Capacidad`. Crear "Mesa Test 1" capacidad 4.
3. Verificá que aparece en la grilla con un `qr_token` visible (16 chars alfanuméricos).
4. Tocá el botón impresora (Printer icon). Debería abrir nueva pestaña en `/print/qr/<token>` con QR + label + auto-print prompt.
5. Volvé a la lista. Tocá editar (Pencil icon). Cambiá nombre a "Mesa Test 1B" + tilde "Mesa activa" desactivar. Guardar.
6. Verificá: nombre actualizado y badge "Inactiva".
7. Reactivar (editar de nuevo).
8. Tocá regenerar (RefreshCw icon). AlertDialog confirmar. Verificá: el `qr_token` cambió.
9. Tocá eliminar (Trash icon). Confirmar. Verificá: desaparece.

### 3.2 Smoke comensal — escanear y registrarse

1. Volver a crear "Mesa 5". Copiar el `qr_token`.
2. Pestaña incógnito: `http://localhost:3000/m/<qr_token>`.
3. Verificá: muestra "Mesa 5" + nombre del bar + spinner mientras se une, después CTA "Registrarme para sumar puntos".
4. Tocá "Registrarme...". Form con: nombre, apellido, teléfono, cumpleaños (opcional), checkbox marketing.
5. Llenar: `Lucia`, `Pérez`, `+5491134567890`, `1990-05-15`, marketing on. Submit.
6. Verificá: toast verde "¡Listo! Tus puntos van a sumarse al cerrar la mesa." y mensaje "Estás registrado para sumar puntos".
7. Refrescá la pestaña. El `browser_token` está en localStorage, la sesión sigue abierta. Estado debería volver a "registrado" automáticamente (la pantalla actualmente solo persiste si se llama join_session_as_guest con el mismo browser_token y register_customer fue exitoso — verificar este detalle de UX).

### 3.3 Smoke bordes

1. **QR inválido**: `http://localhost:3000/m/inventadoXYZ123` → debe mostrar "QR no válido".
2. **Segundo guest en la misma mesa**: pestaña incógnito 2 (browser_token nuevo), escanear el mismo `qr_token` de Mesa 5. Verificá que se une a la sesión existente (guest_count=2 internamente; visualmente se ve "Sesión abierta" igual).
3. **Dedupe de customer**: en la pestaña 2, registrarse con el **mismo teléfono** que Lucia. Verificá:

```bash
psql "$DB_URL" -c "select count(*) from public.session_guests where customer_id is not null;"
# Esperado: 2 (los dos guests apuntando al mismo customer)

psql "$DB_URL" -c "select count(*) from public.customers where phone = '+5491134567890';"
# Esperado: 1
```

## Si algo falla

- **Migración falla**: leer mensaje de error, identificar la migración (`20260506100*`). Si es problema de orden o referencia, revisar `docs/superpowers/specs/2026-05-06-mesas-pedidos-puntos-design.md` §4.
- **Test RLS falla con env vars**: confirmar `.env.local` tiene `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_ANON_KEY` (sin prefijo `NEXT_PUBLIC_`).
- **`/m/<token>` da 500**: verificar service client en `lib/supabase/service.ts` y que los types regenerados (`types/database.ts`) tienen las nuevas tablas.
- **Phone normalizer rechaza tu teléfono**: el helper `tryNormalizePhone` espera formato AR. Probar con `+5491134567890`.
